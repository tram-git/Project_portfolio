
<h2> # Project.2 - 반도체 공정 센서 데이터 기반 공정 최적화 </h2>

### Background
  반도체 공정 센서 데이터를 기반으로 불량 발생을 사전에 예측하고, <br>
  주요 공정 변수와의 관계를 분석하여 불량의 원인을 파악하고 품질 개선 및 공정 최적화 전략을 도출한다. <br> 

  또한 불량률을 낮추는 것은 비용 절감, 제품 신뢰도 향상, 고객 불만 감소, 생산성 증대라는 측면에서 매우 중요하므로 <br>
  머신러닝 예측 모델을 통해 실시간 불량 탐지 체계를 구축할 수 있는 기반을 마련한다.
 
### Summary

  - (1) Data information
    - 8개의 센서 항목이 존재하는 40개 피처로 구성된 반도체 공정 센서 데이터 (총 5개의 공정 과정)
    - 각 공정단계별 주요 변수 (온도, 습도, 유량, 밀도, 점도, 가스농도 편차 등) + 불량 여부를 나타내는 Label 변수

  - (2) Data Preprocessing
    - EDA (데이터 이해 / 불량 여부를 나타내는 Label 확인)
    - 중복값 없음, 결측값 없음, 이상값 유지 (범위가 정해진 편차 데이터이므로 분석에 유의미하다고 판단)
    - Reduction (상관관계가 1.0으로 값이 전부 동일한 변수 3가지 제거)
    - 결측치 확인 및 대체 --> VIF 확인과 병행하여 IterativeImputer (MICE), knn, 평균값 방식으로 결측치 대체

  - (3) Statistics, Visualization Insights
    - 각 변수와 불량 label 변수의 상관관계 확인
    - kde, box, qqplot 등을 통한 변수 분포 확인 
    - 모든 공정의 온도, 습도 데이터가 일정한 분포. 균일 분포 형태임을 확인
    - 머신러닝 모델 학습을 위해 t-test, Mann-Whitney U 검정을 통한 변수 축소 진행

  - (4) model, algorithms
	- 기본 베이스라인 모델을 통한 예측 성능 확인
    - 모델 성능 비교를 통한 불량 탐지 모델 선정 (recall 기준)
    - learning curve를 통한 모델의 과적합 판단

  - (5) Report
    - 반도체 공정 품질 개선 및 공정 최적화를 위한 인사이트 도출
    - 반도체 공정 불량률을 낮추기 위한 머신러닝 모델 선정

  - (6) Feedback
    - 도메인 지식을 활용한 추가 파생 변수가 있었다면 모델 성능을 더 끌어 올릴 가능성 존재.
    - 결론을 비즈니스 언어로 환산하면 보고서의 가치가 훨씬 높아질 수 있다.
    - SHAP 분석 등을 사용했다면, 변수와 타겟 간의 관계를 더 깊이 있게 해석할 수 있다.

  - (7) Review
    - 분석 과정의 판단 근거를 충분히 검토하거나 제안된 아이디어를 직접 실험해보는 데 한계가 있었다.
    - 데이터 분석 단계의 논리적 흐름을 꾸준히 확인할 필요가 있다.
    - 분석 방향성이 한 부분으로 고정되어 있는 경향이 있었으므로, 여러 방향의 의견을 모아 볼 필요를 느꼈다.
    - 문제 상황이 발생했을 때, 의견을 적극적으로 공유하고 방향을 조정하며 프로젝트를 끝까지 마친 부분이 만족스럽다.
