<h2> # Project.1 - 시각화 기반 친환경 SUV 개발 전략 수립 </h2>

### Background
  차량 데이터를 활용해 연비와 배출량의 관계를 분석하고, 이를 기반으로 친환경적 차량 설계 (탄소 배출량 감축) 전략을 제안한다.

### Summary

  - (1) Data information
    - Fuel Consumption 2000-2022, 캐나다 모델별 연료 소비량 등급과 예상 이산화탄소 배출량 데이터
    - 제품 제작 년도, 모델, 클래스, 엔진 사이즈, 연료 종류, 연비, 배출량

  - (2) Data Preprocessing
    - EDA (데이터 이해 / 연비, 배출량을 판단 근거로 사용)
    - Reduction (중복 행 제거 + 동일한 연비 정보를 제공하고 있는 컬럼 제거)

  - (3) Statistics, Visualization Insights
    - 데이터 상에서 2014~2015년 사이 배출량 데이터 값이 급격히 상승
    - --> 2014 캐나다 연료 소모 가이드에 의하면, <br> 2015년부터 차량 배출량과 연비가 10%~20% 정도 높게 책정되었음을 확인, 배출량 값을 15% 하향
    - 실린더 수가 증가함에 따라 탄소 배출량 증가 확인
    - 엔진 사이즈(배기량)가 증가함에 따라 탄소 배출량 증가 확인
    - 무단변속기, 자동변속기의 다단화 / 더블클러치, 수동변속기의 소단화 필요

  - (4) Report
    - 친환경적 차량 설계에 영향을 주는 변수 정리
    - 시각화를 통한 친환경적 차량 설계 인사이트 도출

  - (5) Feedback
    - 데이터를 그대로 포함하거나 어려운 용어 사용을 지양해야 한다.
    - 분석 과정을 복잡하게 작성하면 청자의 이해에 어려움을 겪을 수 있다.

  - (6) Review
    - 시간 분배와 일정 수립 과정에서 많은 시행착오 발생.
    - 데이터를 있는 그대로 (결론을 정해두지 않고) 받아들이는 연습이 되었다.
    - 데이터에서 확인할 수 없는 외부 요인을 발견하는 좋은 경험을 하였다.

[시각화 기반 친환경 SUV 개발 전략 수립] (https://github.com/tram-git/Project_portfolio/tree/main/SUV%20development%20strategy)<br>

<h2> # Project.2 - 반도체 공정 센서 데이터 기반 공정 최적화 </h2>

### Background
  반도체 공정 센서 데이터를 기반으로 불량 발생을 사전에 예측하고, <br>
  주요 공정 변수와의 관계를 분석하여 불량의 원인을 파악하고 품질 개선 및 공정 최적화 전략을 도출한다. <br> 

  또한 불량률을 낮추는 것은 비용 절감, 제품 신뢰도 향상, 고객 불만 감소, 생산성 증대라는 측면에서 매우 중요하므로 <br>
  머신러닝 예측 모델을 통해 실시간 불량 탐지 체계를 구축할 수 있는 기반을 마련한다.
 
### Summary

  - (1) Data information
    - 8개의 센서 항목이 존재하는 40개 피처로 구성된 반도체 공정 센서 데이터 (총 5개의 공정 과정)
    - 각 공정단계별 주요 변수 (온도, 습도, 유량, 밀도, 점도, 가스농도 편차 등) + 불량 여부를 나타내는 Label 변수

  - (2) Data Preprocessing
    - EDA (데이터 이해 / 불량 여부를 나타내는 Label 확인)
    - 중복값 없음, 결측값 없음, 이상값 유지 (범위가 정해진 편차 데이터이므로 분석에 유의미하다고 판단)
    - Reduction (상관관계가 1.0으로 값이 전부 동일한 변수 3가지 제거)
    - 결측치 확인 및 대체 --> VIF 확인과 병행하여 IterativeImputer (MICE), knn, 평균값 방식으로 결측치 대체

  - (3) Statistics, Visualization Insights
    - 각 변수와 불량 label 변수의 상관관계 확인
    - kde, box, qqplot 등을 통한 변수 분포 확인 
    - 모든 공정의 온도, 습도 데이터가 일정한 분포. 균일 분포 형태임을 확인
    - 머신러닝 모델 학습을 위해 t-test, Mann-Whitney U 검정을 통한 변수 축소 진행

  - (4) model, algorithms
	- 기본 베이스라인 모델을 통한 예측 성능 확인
    - 모델 성능 비교를 통한 불량 탐지 모델 선정 (recall 기준)
    - learning curve를 통한 모델의 과적합 판단

  - (5) Report
    - 반도체 공정 품질 개선 및 공정 최적화를 위한 인사이트 도출
    - 반도체 공정 불량률을 낮추기 위한 머신러닝 모델 선정

  - (6) Feedback
    - 도메인 지식을 활용한 추가 파생 변수가 있었다면 모델 성능을 더 끌어 올릴 가능성 존재.
    - 결론을 비즈니스 언어로 환산하면 보고서의 가치가 훨씬 높아질 수 있다.
    - SHAP 분석 등을 사용했다면, 변수와 타겟 간의 관계를 더 깊이 있게 해석할 수 있다.

  - (7) Review
    - 분석 과정의 판단 근거를 충분히 검토하거나 제안된 아이디어를 직접 실험해보는 데 한계가 있었다.
    - 데이터 분석 단계의 논리적 흐름을 꾸준히 확인할 필요가 있다.
    - 분석 방향성이 한 부분으로 고정되어 있는 경향이 있었으므로, 여러 방향의 의견을 모아 볼 필요를 느꼈다.
    - 문제 상황이 발생했을 때, 의견을 적극적으로 공유하고 방향을 조정하며 프로젝트를 끝까지 마친 부분이 만족스럽다.

<h2> # Project.3 - 줄기세포 분화 가능성 탐지 </h2>

### Background
  시계열 특성을 지닌 줄기세포 분화 공정 데이터를 분석하여, <br>
  최적의 분화 성공률을 달성하기 위한 최적 분화 공정 조건을 탐색한다. <br> 

  '분화 성공' 그룹과 '분화 실패' 그룹 간의 공정 변수 값에 유의미한 차이가 있는지 검증하고 <br>
  실패 배치를 높은 재현율(Recall)로 잡아낼 수 있는 예측 모델을 구축한다.<br> 
  또한 대시보드를 구성하여 모델을 기반으로 한 '조기 경보 시스템' 구축 방안을 제안한다.<br> 
  
  결과적으로 최소한의 초기 데이터만으로 최종 결과를 예측하여, 연구개발 비용과 시간을 절약하는 것을 목적으로 한다.
  
### Summary

  - (1) Data information
    - 시계열 특성을 지닌 줄기세포 분화 공정 데이터 (0 ~ 7일차)
    - 주요 변수 값 (밀도, 응집 크기, 전배양, 용존 산소, pH, 젖산, 포도당) + 성공 여부를 확인할 수 있는 10일차 결과 변수
    - 각 일자 변화량 값 (밀도, 응집 크기, 용존 산소, pH의 변화량 값)

  - (2) Data Preprocessing
    - EDA (데이터 이해 / 10일차 결과 변수 값 70 이상이면 성공 판정)
    - 10일차 결과 변수를 70 기준으로 하여 "Success/Failure" 컬럼 (0, 1) 생성
    - 중복값 없음, 이상값 유지 (60개 행의 적은 데이터로 인해 이상값 판단 불가)
    - 모든 값이 결측치인 변수 18개 확인 --> 해당 변수값을 판단할 수 있는 정보가 없어 제거

  - (3) Statistics, Visualization Insights
    - 각 변수와 10일차 결과 변수와의 상관관계 확인
    - 각 변수 분포 확인 및 성공 / 실패 그룹 비교
    - 가설 설정 및 검증을 통한 분화 성공 영향의 변수 별 방향성 파악
    - 각 시각화 자료를 뒷받침할 통계 분석 진행

  - (4) model, algorithms
	- 기본 베이스라인 모델을 통한 예측 성능 확인
    - SMOTE, Gridsearch를 통한 모델 예측 성능 상승 유도
    - 모델 성능 비교를 통한 불량 탐지 모델 선정 (recall 기준)
    - 혼동 행렬 확인 및 SHAP 분석과 feature importance 비교
   
  - (5) dashboard
    - 각 일차 (0 ~ 7일 차) 조정 사이드바 구성, 일차별 동적인 예측 결과 표현
    - KPI 페이지 - 전체 수, 정상 예측 비율, 이상 예측 비율, 주의 샘플 수를 KPI 지표로 제작
    - 예측 결과 페이지 - 전체 샘플 예측 결과와 권장 조치 내역 카드 구성
    - 이상 원인 분석 페이지 - 이상에 기여한 순서대로 변수 요약 및 바 차트
    - 데이터 분포 페이지 - 성공 / 실패 예측 그룹 간 데이터 분포를 라인, 박스 차트로 비교
 
  - (6) Report
    - 시각화, 통계 분석을 통한 설정 가설 검증 및 분화 공정 조건 정리
    - 불량률을 낮추고 비용과 시간을 절약하기 위한 머신러닝 모델 선정
    - 실패 여부를 조기에 확인하고 예방할 수 있는 대시보드 구현 

  - (7) Feedback
    - 도메인 그룹과 측정 간격 불규칙성(시계열 특성)을 결론부가 아닌 EDA 초반에 제시하였어야 한다.
    - '예측·분류·이상탐지'의 혼재로, 목적을 명확히 선언하고 그에 맞는 지표와 모델 근거를 일관되게 표현하여야 한다.
    - 도메인 문헌에 근거한 5개 가설을 체계적으로 검증하고, <br> SHAP·기여도 기반의 대시보드로 연결하여 바이오 공정의 특성을 잘 보여 준다.
  
  - (8) Review
    - 현 머신러닝 모델은 제한된 데이터 수로 인해 만족스러운 recall 값을 갖지 못하는 상태.
    - 공정 차원에서 현재 모델은 실패 샘플을 걸러 내는 일시적인 용도로 사용하여야 한다.
    - 줄기세포 분화 데이터가 쌓이면 새로운 모델을 형성하고 교체하는 것이 현 상황에서 자연스러운 흐름으로 보인다.
    - 모델의 완성도는 아쉬우나, 시계열 특성을 가진 데이터를 각 일자별로 예측해 보았던 경험이 만족스럽다.
